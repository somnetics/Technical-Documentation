  
# Seamless Platform Technical Documentation

### Folder structure

Here's a folder structure for a Seamless Platform:

```
seamless-platform/  # Root directory.
|- app                      # Directory for application related file
  |- apis                   # Directory used to store builded (output) files.
  |- routes                 # Directory used to store builded (output) files.
  |- services               # Directory used to store builded (output) files.
  |- templates              # Directory used to store builded (output) files.
  |- views                  # Directory used to store builded (output) files.
    |- partials             # Directory used to store builded (output) files.
|- core                     # Directory used to store builded (output) files.
  |- handlers               # Directory used to store builded (output) files.
  |- libs                   # Directory used to store builded (output) files.
  |- services               # Directory used to store builded (output) files.
|- public                   # Directory used to store builded (output) files.
```

Here's a folder structure for a Seamless Platform:

```
|- app                      # Directory for application related file
  |- apis                   # Directory for application related file
  |- configs                # Directory for application related file
  |- flows                  # Directory for application related file
  |- forms                  # Directory for application related file
  |- licenses               # Directory for application related file
  |- privileges             # Directory for application related file
  |- reports                # Directory for application related file
  |- routes                 # Directory for application related file
  |- services               # Directory for application related file
  |- templates              # Directory for application related file
  |- views                  # Directory for application related file
|- cache                    # Directory used to store builded (output) files.
|- routes                   # Directory used to store builded (output) files.
|- config                   # Directory used to store builded (output) files.
|- libs                     # Directory used to store builded (output) files.
|- locales                  # Directory used to store builded (output) files.
|- public                   # Directory used to store builded (output) files.
|- session                  # Directory used to store builded (output) files.

```
